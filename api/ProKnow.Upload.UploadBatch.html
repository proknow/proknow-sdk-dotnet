Creates a batch of resolved uploads

public UploadBatch(ProKnowApi proKnow, string workspaceId, IList<UploadProcessingResult> uploadProcessingResults)

Parameters

proKnow ProKnowApi

The root object for interfacing with the ProKnow API

workspaceId string

The ProKnow ID of the workspace

uploadProcessingResults IList<UploadProcessingResult>

The upload processing results

Examples

This example shows how to create an upload batch: 

using ProKnow;
using System.IO;
using System.Threading.Tasks;

 var pk = new ProKnowApi("https://example.proknow.com", "credentials.json");
 var workspaceItem = await pk.Workspaces.ResolveByNameAsync("Upload Test");
 var uploadResults = await pk.Uploads.UploadAsync(workspaceItem, "./DICOM");
 var uploadProcessingResults = await pk.Uploads.GetUploadProcessingResultsAsync(workspaceItem, uploadResults);
 var uploadBatch = new UploadBatch(pk, workspaceItem.Id, uploadProcessingResults.Results);</code></pre>

GetStatus(string)

Get the processing status for a provided path

public string GetStatus(string path)

Parameters

path string

The full path to the file

Returns

string

"completed" if the upload processing was successful, "pending" if the upload was successful but conflicts occurred during processing and it needs attention, or "failed" if the upload could not be processed.

Examples

This example shows how to get the processing status for a given file upload: 

using ProKnow;
using System.IO;
using System.Threading.Tasks;

 var pk = new ProKnowApi("https://example.proknow.com", "credentials.json");
 var workspaceItem = await pk.Workspaces.ResolveByNameAsync("Upload Test");
 var uploadResults = await pk.Uploads.UploadAsync(workspaceItem, "./DICOM");
 var uploadProcessingResults = await pk.Uploads.GetUploadProcessingResultsAsync(workspaceItem, uploadResults);
 var uploadBatch = new UploadBatch(pk, workspaceItem.Id, uploadProcessingResults);
 var path = Path.GetFullPath(Path.Join("./DICOM", "reg.dcm"));
 var status = uploadBatch.GetStatus(path);</code></pre>
