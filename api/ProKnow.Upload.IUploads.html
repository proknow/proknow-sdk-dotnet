GetUploadProcessingResultsAsync(WorkspaceItem, IList<UploadResult>)

Gets the processing results for a provided set of uploads

Task<UploadProcessingResults> GetUploadProcessingResultsAsync(WorkspaceItem workspace, IList<UploadResult> uploadResults)

Parameters

workspace WorkspaceItem

The ProKnow ID for the workspace

uploadResults IList<UploadResult>

The upload results for each file

Returns

Task<UploadProcessingResults>

The processing results for each file

Examples

This example shows how to get the processing results for an uploaded directory of files: 

using ProKnow;
using System.Threading.Tasks;

 var pk = new ProKnowApi("https://example.proknow.com", "credentials.json");
 var workspaceItem = await pk.Workspaces.ResolveByNameAsync("Upload Test");
 var uploadResults = await pk.Uploads.UploadAsync(workspaceItem, "./DICOM");
 var uploadProcessingResults = await pk.Uploads.GetUploadProcessingResultsAsync(workspaceItem, uploadResults);</code></pre>

Remarks

The Results property of the return value of this method can be used to construct an UploadBatch which can be used to look up patients, entities, spatial registration objects, and file statuses within the processing results.

This overload is more performant than the one that takes a string workspace ProKnow ID or name because the workspace does not need to be resolved.

UploadAsync(WorkspaceItem, IList<string>, UploadFileOverrides)

Upload files asynchronously

Task<IList<UploadResult>> UploadAsync(WorkspaceItem workspaceItem, IList<string> paths, UploadFileOverrides overrides = null)

Parameters

workspaceItem WorkspaceItem

The workspace

paths IList<string>

The folder and/or file paths

overrides UploadFileOverrides

Optional overrides to be applied after the files are uploaded

Returns

Task<IList<UploadResult>>

The upload results

Examples

This example shows how to upload a directory of files: 

using ProKnow;
using System.Collections.Generic;
using System.IO;
using System.Threading.Tasks;

 var pk = new ProKnowApi("https://example.proknow.com", "credentials.json");
 var paths = new List<string>() {
     Path.Join("DICOM", "img000001.dcm"),
     Path.Join("DICOM", "img000002.dcm"),
     Path.Join("DICOM", "img000003.dcm"),
     Path.Join("DICOM", "img000004.dcm"),
     Path.Join("DICOM", "img000005.dcm")
 }
 var workspaceItem = await pk.Workspaces.ResolvedByNameAsync("Upload Test");
 var uploadResults = await pk.Uploads.UploadAsync(workspaceItem, paths);</code></pre>

Lists containing both directories and file paths are also permitted: 

using ProKnow;
using System.Collections.Generic;
using System.IO;
using System.Threading.Tasks;

 var pk = new ProKnowApi("https://example.proknow.com", "credentials.json");
 var paths = new List<string>() {
     Path.Join("DICOM", "CT"),
     Path.Join("DICOM", "structures.dcm"),
     Path.Join("DICOM", "plan.dcm"),
     Path.Join("DICOM", "dose.dcm")
 }
 var workspaceItem = await pk.Workspaces.ResolvedByNameAsync("Upload Test");
 var uploadResults = await pk.Uploads.UploadAsync(workspaceItem, paths);</code></pre>

Remarks

NOTE that this task completes when the upload itself has finished. ProKnow still needs to process each object uploaded before it becomes available for use. GetUploadProcessingResultsAsync(WorkspaceItem, IList<UploadResult>) to obtain the processing results.

This overload is more performant than the one that takes a string workspace ProKnow ID or name because the workspace does not need to be resolved.
